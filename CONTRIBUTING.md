# Внесок у NEXOLAB

## Робочий процес

1. Створіть гілку від `main`: `feat/...`, `fix/...`, `docs/...` або `chore/...`.
2. Використовуйте Conventional Commits, наприклад `feat: add node health panel`.
3. Перед push виконайте локальні перевірки.
4. Відкрийте Pull Request і заповніть шаблон.
5. Візуальні зміни мають містити скриншот desktop і mobile станів.

## Definition of Done

- TypeScript не містить помилок.
- ESLint і Prettier проходять без попереджень.
- Нову логіку покрито тестами на релевантному рівні.
- UI має keyboard focus, доступні назви та не покладається лише на колір.
- Компоненти не містять секретів або production credentials.
- Документацію оновлено, якщо змінилася поведінка або архітектура.

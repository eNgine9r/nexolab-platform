# Security Policy

## Повідомлення про вразливість

Не створюйте публічний issue для вразливостей, витоків ключів або способів обходу авторизації. Надішліть приватне повідомлення власнику репозиторію через GitHub profile contact.

## Базові правила

- Секрети зберігаються лише у змінних середовища.
- `.env` файли не комітяться.
- Керування замками, реле та небезпечними режимами вимагає server-side RBAC, повторного підтвердження та audit log.
- Дані інтерфейсу не вважаються довіреними; backend має повторно перевіряти всі команди.
- Dependabot і CI повинні залишатися увімкненими.
